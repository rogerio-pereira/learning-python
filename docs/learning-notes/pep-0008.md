# PEP 8 (Guia de Estilo para Código Python)

Esta é uma tradução livre e resumida da PEP 8.  
Para o texto oficial, consulte: https://peps.python.org/pep-0008/

---

## Objetivo

A PEP 8 define um conjunto de boas práticas de estilo para tornar o código Python consistente, legível e fácil de manter.

---

## Princípios centrais (em linguagem simples)

- Legibilidade importa mais do que "código curto".
- Consistência dentro do projeto é mais importante do que preferências pessoais.
- Código deve ser simples de ler "batendo o olho".

---

## Regras práticas mais importantes

### Indentação e linhas
- Use 4 espaços por nível de indentação.
- Evite linhas muito longas (o padrão sugere 79 caracteres para código).

### Espaçamento
- Use um espaço ao redor de operadores: `a + b`, `x == y`.
- Não coloque espaços dentro de parênteses: `func(arg)` e não `func( arg )`.

### Nomes
- Funções e variáveis: `snake_case`.
- Classes: `PascalCase`.
- Constantes: `UPPER_SNAKE_CASE`.

### Imports
- Um import por linha.
- Preferir imports explícitos e organizados por grupos (stdlib, terceiros, projeto).

### Linhas em branco
- Use linhas em branco para separar blocos lógicos.
- Separe funções de alto nível com duas linhas em branco.

---

## Quando quebrar a regra

Se uma regra torna o código menos legível, prefira legibilidade.  
Mas mantenha consistência com o time e o projeto.
