# PEP 257 (Docstrings)

Esta é uma tradução livre e resumida da PEP 257.  
Para o texto oficial, consulte: https://peps.python.org/pep-0257/

---

## Objetivo

Padronizar docstrings (documentação interna) para que pessoas e ferramentas entendam o que módulos, classes e funções fazem.

---

## Ideia principal

Docstring é a primeira string literal dentro de um módulo, classe ou função.  
Ela descreve o propósito do elemento, não a implementação detalhada.

---

## Boas práticas essenciais

### Docstrings de uma linha

Use quando a explicação é simples.

```python
def calculate_total(price, quantity):
    """Return the total price."""
    return price * quantity
```

### Docstrings multilinha

Use quando precisa de contexto maior.

```python
def calculate_total(price, quantity):
    """Return the total price.

    The result is computed as price * quantity.
    """
    return price * quantity
```

### Onde usar
- Módulos: o que este arquivo representa.
- Classes: o que esta classe representa.
- Funções/métodos: o que fazem e o que retornam.

---

## O que evitar

- Não repita o óbvio.
- Não descreva linha por linha.
- Evite docstrings que não agregam clareza.
