# PEP 484 (Type Hints)

Esta é uma tradução livre e resumida da PEP 484.  
Para o texto oficial, consulte: https://peps.python.org/pep-0484/

---

## Objetivo

A PEP 484 define o sistema de type hints em Python: uma forma de anotar tipos sem mudar o comportamento em runtime.

---

## Ideia principal

Type hints servem para:
- documentar intenções,
- ajudar ferramentas como mypy,
- melhorar autocomplete e leitura do código.

Eles não forçam o tipo em tempo de execução por padrão.

---

## Exemplos básicos

```python
def greet(name: str) -> str:
    return f"Hello, {name}"
```

Aqui você declara que `name` deve ser `str` e que o retorno é `str`.

---

## Coleções e tipos compostos

```python
from typing import List, Optional

def average(values: List[int]) -> Optional[float]:
    if not values:
        return None
    return sum(values) / len(values)
```

- `List[int]`: lista de inteiros.
- `Optional[float]`: `float` ou `None`.

---

## Tipos personalizados

```python
class Motorcycle:
    def __init__(self, model: str) -> None:
        self.model = model
```

Aqui o construtor recebe `model` como `str` e não retorna nada (`None`).

---

## O que a PEP 484 não faz

- Não transforma Python em linguagem estaticamente tipada.
- Não muda o runtime por padrão.
- Não substitui testes ou validações de entrada.
